# Guia de Contribuição

Obrigado por estar interessado em contribuir com este projeto! Aqui estão algumas diretrizes.

## Como Contribuir

### 1. Issues

Se você encontrar um bug ou tiver uma sugestão:
- Abra uma [issue](../../issues)
- Descreva claramente o problema ou a sugestão
- Forneça exemplos reproduzíveis se possível

### 2. Pull Requests

1. Faça um Fork do repositório
2. Crie uma branch para sua feature (`git checkout -b feature/AmazingFeature`)
3. Commit suas mudanças (`git commit -m 'Add some AmazingFeature'`)
4. Push para a branch (`git push origin feature/AmazingFeature`)
5. Abra um Pull Request

### 3. Código e Estilo

- Use formatação padrão PEP 8
- Adicione docstrings em funções novas
- Inclua testes quando apropriado
- Atualize a documentação se necessário

### 4. Sugestões de Melhorias

Ideias para o projeto:

#### Curto Prazo
- [ ] Melhorar visualizações
- [ ] Adicionar testes unitários
- [ ] Documentação em outras linguagens
- [ ] Exemplo com dados próprios

#### Médio Prazo
- [ ] Análise de ciclos de ordem superior
- [ ] Suporte para grafos ponderados
- [ ] Performance optimizations
- [ ] Publicar dados como dataset

#### Longo Prazo
- [ ] Biblioteca Python independente
- [ ] Comparação com outras métricas topológicas
- [ ] Análise de grafos dinâmicos
- [ ] Aplicações em machine learning

## Padrão de Commits

Use mensagens claras em commit:

```
[tipo] Descrição breve

Descrição mais detalhada se necessário.

- Ponto 1
- Ponto 2
```

Tipos: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

Exemplos:
- `feat: Add VIF analysis for multicollinearity`
- `fix: Correct triangle counting algorithm`
- `docs: Update mathematical background`

## Código de Conduta

- Seja respeitoso
- Aceite crítica construtiva
- Foque no que é melhor para a comunidade

## Dúvidas?

Abra uma issue com a tag `question`.
